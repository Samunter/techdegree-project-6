FORM BASICS

THE FORM ELEMENT
Cannot nest form inside a form.

The action attribute is the web address of a program that processes the information submitted via the form.

The method attribute is the HTTP method that the browser uses to submit the form. We'll usually use "post"
- "post" data from the body of the form is sent, or posted, to the server
- "get" data is sent inside of the URL, and parameters are separated with a question mark (http://example.com?user_name=Nick)

ex:
<form action="index.html" method="post"></form>






THE INPUT ELEMENT

self-closing tag, doesn't need a closing tag

many different types

ex:
<input type="text" id="name" name="user_name">
<input type="email" id="mail" name="user_email">
<input type="password" id="password" name="user_password">






THE TEXTAREA ELEMENT

for multiline inputs, like a bio

ex:
<textarea id="bio" name="user_bio"></textarea>






THE BUTTON ELEMENT

should also be nested inside the form element

multiple button types:
-submit sends data to the web server
-reset automatically clears all form data when clicked
-button has no default behavior, intended to be used with js















HTML FORMS

THE LABEL ELEMENT

"for" attribute matches to "id" - when the label is clicked, the associated form will be selected

ex:
<label for="name">Name:</label>
<input type="text" id="name" name="user_name">






FIELDSETS AND LEGENDS

Fieldset is for logically grouping forms

Legends are labels/titles for fieldsets

ex:

<fieldset>
   <legend><span class="number">2</span> Your profile</legend>
   
   <label for="bio">Biography:</label>
   <textarea id="bio" name="user_bio"></textarea>

</fieldset>















CHOOSING OPTIONS

SELECT MENUS

Best to use the select element if you have more than 4-5 options, saves space

Needs option elements inside of it, just like an ul element needs li elements 

Can group option elements with optgroup element

ex:

<select id="job" name="user_job">
   <optgroup label="Web">
      <option value="web_designer">Web Designer</option>
   </optgroup>
</select>






RADIO BUTTONS

Only one radio button in a group may be selected at a time. Groups are based on the "name" attribute

The label for the radio button area will not have a "for" attribute, because there is not one element to associate it with

ex:
<label>Age:</label>
   <input type="radio" id="under_13" value="under_13" name="user_age"><label for="under_13" class="light">Under 13</label><br>
   <input type="radio" id="over_13" value="over_13" name="user_age"><label for="over_13" class="light">13 or Older</label>


CHECKBOXES

May select more than one

ex:

<label>Interests:</label>
   <input type="checkbox" id="development" value="interest_development" name="user_interest"><label class="light" for="development">Development</label><br>
